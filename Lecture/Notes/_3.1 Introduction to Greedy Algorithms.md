# 1. Real World Motivation

Imagine you are shopping with ₹1000.

You want to buy as many useful items as possible.

You enter the store.

Instead of checking every possible combination of items, you simply pick the cheapest useful item first.

Then again.

Then again.

Every time you make **the best decision available at that moment.**

This is exactly how a Greedy algorithm works.

It never looks back.

It never changes previous decisions.

# 3. What is a Greedy Algorithm?

## Definition

A Greedy Algorithm is an algorithmic technique that builds the solution step by step by **always choosing the locally optimal choice**, hoping that these local choices will lead to the globally optimal solution.

---

## Simple Definition

> Make the best decision now.
>
> Never reconsider it later.

---

# 4. Important Characteristics

A Greedy algorithm has these properties.

## 1. Step-by-Step Construction

The solution is built gradually.

Example

```
Choose first edge

Choose second edge

Choose third edge
```

instead of generating the entire solution at once.

---

## 2. Local Optimal Choice

Every decision is the best decision **at that moment**.

Not necessarily the overall best.

---

## 3. Irreversible Decisions

Once selected,

it is never removed.

Example

```
Pick edge

Cannot undo

Move forward
```

Unlike Dynamic Programming.

---

## 4. Fast Execution

Since previous decisions are never revisited,

Greedy algorithms are usually very fast.

Often

```
O(n)

O(n log n)
```

---

# 5. General Structure of Greedy Algorithm

```
Start

↓

Find all available choices

↓

Select best choice

↓

Add into solution

↓

Remove impossible choices

↓

Repeat until complete
```

---

# 6. Generic Greedy Algorithm

```
Initialize Solution

while(problem not solved)

    Find best candidate

    if(candidate is feasible)

         Select candidate

Return Solution
```

```text

      Find Best Item

      if(item is valid)

            Add item

return Solution
```

# 8. Why is it Called "Greedy"?

Because it always wants the **best immediate reward.**

It does not think about future consequences.

Just like a greedy person who always grabs the biggest available piece of cake.

---

# 9. Local Optimum vs Global Optimum

This is the MOST IMPORTANT concept.

---

## Local Optimum

Best choice at current step.

---

## Global Optimum

Best possible overall solution.

---

### Example

Suppose destination is

```
A → B → C
```

At A

Road 1 costs 5

Road 2 costs 6

Greedy chooses

```
5
```

because it is locally best.

But later

```
Road 1 leads to expensive route.

Road 2 leads to cheap route.
```

Finally

```
Road 1 total = 100

Road 2 total = 30
```

Greedy failed.

---

# 10. When Does Greedy Work?

Greedy works **only if** the problem satisfies **both** of the following properties:

## Property 1: Greedy Choice Property

A globally optimal solution can be reached by making a locally optimal choice at each step.

In simple words:

> Choosing the best option now should still allow reaching the best overall answer.

---

## Property 2: Optimal Substructure

An optimal solution to the whole problem contains optimal solutions to its subproblems.

Example:

Shortest path

↓

Shortest path to destination contains shortest path to intermediate nodes.

---

If either property is missing, Greedy may fail.

# 16. Java Example (Simple Greedy: Largest Coin First)

```java
public class GreedyCoin {

    public static void main(String[] args) {

        int[] coins = {50, 20, 10, 5, 2, 1};
        int amount = 87;

        System.out.println("Coins used:");

        for (int coin : coins) {
            while (amount >= coin) {
                System.out.print(coin + " ");
                amount -= coin;
            }
        }
    }
}
```

**Output:**

```text
Coins used:
50 20 10 5 2
```

# 17. C++ Example (Simple Greedy: Largest Coin First)

```cpp
#include <iostream>
using namespace std;

int main() {
    int coins[] = {50, 20, 10, 5, 2, 1};
    int amount = 87;

    cout << "Coins used: ";

    for (int coin : coins) {
        while (amount >= coin) {
            cout << coin << " ";
            amount -= coin;
        }
    }

    return 0;
}
```

**Output:**

```text

```

# 20. Common Mistakes

- Assuming greedy always finds the optimal solution.
- Applying greedy without verifying the Greedy Choice Property.
- Forgetting to sort input when the algorithm requires sorted data.
- Ignoring feasibility checks before selecting an option.
- Confusing local optimum with global optimum.

---

# 21. Best Practices

- Verify both **Greedy Choice Property** and **Optimal Substructure** before designing a greedy solution.
- Clearly define the selection criterion (minimum cost, maximum profit, earliest finish time, etc.).
- Use efficient data structures such as priority queues or sorting when appropriate.
- Test the algorithm with edge cases and known counterexamples.
- Compare the greedy approach with brute force or dynamic programming for validation during problem solving.

# 24. Summary

- A Greedy Algorithm constructs a solution incrementally by making the best local decision at each step.
- Decisions are **irreversible**; once made, they are not changed.
- Greedy algorithms are generally simple, memory-efficient, and fast.
- They work correctly only when the problem satisfies the **Greedy Choice Property** and **Optimal Substructure**.
- Classic greedy applications include **Minimum Spanning Tree (Prim's and Kruskal's), Dijkstra's Algorithm, Fractional Knapsack, Activity Selection, and Huffman Coding**.
- Greedy is a powerful design technique, but it must be applied with careful correctness analysis because it does not guarantee an optimal solution for every problem.
