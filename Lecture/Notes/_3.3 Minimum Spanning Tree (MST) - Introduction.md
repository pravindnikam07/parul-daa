# 1. Real-World Motivation

Imagine the government wants to connect **five villages** using electric cables.

Each connection has a different installation cost.

Example:

```text
Village A → Village B = ₹5 lakh
Village A → Village C = ₹8 lakh
Village B → Village C = ₹2 lakh
Village C → Village D = ₹6 lakh
Village D → Village E = ₹3 lakh
Village B → Village E = ₹10 lakh
```

The goal is:

- Connect **all villages**
- Spend **minimum money**
- Avoid unnecessary cable loops

This is exactly the **Minimum Spanning Tree (MST)** problem.

# 3. Revision: What is a Graph?

A **Graph** is a mathematical structure used to represent relationships between objects.

A graph consists of:

- **Vertices (Nodes)** → Objects
- **Edges** → Connections

Example:

```text
      A
     / \
    B---C
```

Here,

Vertices

```text
A, B, C
```

Edges

```text
AB
AC
BC
```

# 4. Types of Graphs

## 4.1 Undirected Graph

Edges have **no direction**.

```text
A ----- B
```

Means

A is connected to B

AND

B is connected to A

MST is always defined on **undirected graphs**.

---

## 4.2 Directed Graph

Edges have direction.

```text
A -----> B
```

Meaning

Only A can reach B through this edge.

**MST is NOT defined for directed graphs** in its standard form.

---

# 5. Weighted Graph

A graph in which every edge has a weight (or cost).

Example:

```text
        4
   A -------- B
   |          |
  2|          |5
   |          |
   C----------D
        3
```

Numbers represent:

- Cost
- Distance
- Time
- Length
- Network latency

depending on the problem.

---

# Classroom Discussion

**Question:**
Can we find an MST in an unweighted graph?

**Answer:**
If all edges have the same weight, **any spanning tree** is also a minimum spanning tree because all spanning trees have the same total cost.

---

# 6. What is a Tree?

A **Tree** is a connected graph that has **no cycles**.

Properties:

- Connected
- No cycles
- For **n vertices**, it has exactly **n − 1 edges**

Example:

```text
      A
     / \
    B   C
         \
          D
```

Vertices = 4

Edges = 3

Formula:

```text
Edges = Vertices − 1
```

---

# 7. What is a Cycle?

A cycle is a path that starts and ends at the same vertex.

Example:

```text
A ----- B
|       |
|       |
D ----- C
```

Cycle

```text
A → B → C → D → A
```

Trees **never** contain cycles.

# 8. What is a Spanning Tree?

A **Spanning Tree** is a tree that:

- Includes **all vertices** of the graph.
- Is connected.
- Has **no cycles**.
- Contains exactly **n − 1 edges**.

---

## Example

Original Graph

```text
        2
    A ------- B
    |\       /|
    | \5   /3 |
   4|  \ /    |
    |   X     |
    | /   \   |
    |/     \  |
    C-------D
        1
```

One possible spanning tree:

```text
A------B
|
|
C------D
```

Another spanning tree:

```text
A
|
|
C------D
 \
  \
   B
```

Both connect all vertices.

Both have:

```text
Vertices = 4

Edges = 3
```

---

# Important Observation

A graph can have **many different spanning trees**.

Question:

Which one is the best?

Answer:

The one with the **minimum total edge weight**.

This leads to MST.

---

# 9. What is a Minimum Spanning Tree (MST)?

## Definition

A **Minimum Spanning Tree** is a spanning tree whose **total edge weight is minimum** among all possible spanning trees.

---

## Example

Consider the graph:

```text
        4
    A ------- B
    | \       |
   2|  \5     |6
    |   \     |
    C ------- D
         3
```

Possible Spanning Tree 1

```text
AB = 4

AC = 2

CD = 3

Total = 9
```

Possible Spanning Tree 2

```text
AB = 4

BD = 6

AC = 2

Total = 12
```

Minimum cost is

```text
9
```

Hence,

Tree 1 is the MST.

# 11. Properties of Minimum Spanning Tree

## Property 1

Contains **all vertices**.

---

## Property 2

Has exactly

```text
V − 1
```

edges.

---

## Property 3

Contains **no cycles**.

---

## Property 4

Has minimum possible total edge weight.

---

## Property 5

If all edge weights are distinct,

the MST is **unique**.

Example:

```text
1

2

3

4

5
```

Only one MST exists.

If some edge weights are equal, multiple MSTs with the same total weight may exist.

---

# 12. Why Avoid Cycles?

Suppose we have:

```text
A------B
|      |
|      |
D------C
```

If all four edges remain, one edge is redundant.

Removing any one edge:

- Keeps all vertices connected.
- Reduces total cost.
- Eliminates the cycle.

Therefore, an MST never contains cycles.

---

# 13. Why Greedy Works for MST

At each step:

- Choose the **smallest safe edge**.
- Add it if it **does not create a cycle**.
- Repeat until **V − 1 edges** are selected.

This local decision leads to the global optimum because MST satisfies the **Greedy Choice Property** and **Optimal Substructure**.

This principle forms the basis of:

- Kruskal's Algorithm
- Prim's Algorithm

---

# 14. Difference Between MST and Shortest Path

Many students confuse these concepts.

| Minimum Spanning Tree         | Shortest Path                                                     |
| ----------------------------- | ----------------------------------------------------------------- |
| Connects all vertices         | Finds shortest path between source and destination                |
| Total graph cost is minimized | Distance from one source is minimized                             |
| No cycles                     | Cycles may exist in the graph (but not in the shortest path tree) |
| Uses V−1 edges                | Number of edges depends on the paths                              |
| Algorithms: Prim, Kruskal     | Algorithms: Dijkstra, Bellman-Ford                                |

---

### Example

Suppose cities are connected by roads.

**Shortest Path Problem:**

"Find the shortest route from Ahmedabad to Surat."

Only one destination matters.

**MST Problem:**

"Lay optical fiber connecting all cities using the least total cable."

Every city must be connected.

# 16. Dry Run (Manual MST Selection)

Consider the graph:

```text
      A
     / \
   2/   \5
   B-----C
    \4  /
     \ /
      D
      |
      1
      E
```

Edge List:

| Edge | Weight |
| ---- | -----: |
| DE   |      1 |
| AB   |      2 |
| CD   |      3 |
| BD   |      4 |
| AC   |      5 |

**Step 1:** Select DE (weight 1)
**Step 2:** Select AB (weight 2)
**Step 3:** Select CD (weight 3)
**Step 4:** Select BD (weight 4) — connects all components without forming a cycle.

Selected edges = 4 = V − 1 (5 − 1)

Total cost = **1 + 2 + 3 + 4 = 10**

# 17. Java Helper Code (Graph Representation)

```java
import java.util.*;

class Edge {
    int source, destination, weight;

    Edge(int source, int destination, int weight) {
        this.source = source;
        this.destination = destination;
        this.weight = weight;
    }
}

public class GraphRepresentation {

    public static void main(String[] args) {

        List<Edge> edges = new ArrayList<>();

        edges.add(new Edge(0, 1, 4));
        edges.add(new Edge(0, 2, 2));
        edges.add(new Edge(1, 2, 5));
        edges.add(new Edge(2, 3, 3));

        for (Edge edge : edges) {
            System.out.println(edge.source + " -> " +
                               edge.destination +
                               " Weight = " + edge.weight);
        }
    }
}
```

# Common Mistakes

- Confusing **Tree** with **Spanning Tree**.
- Assuming every graph has only one spanning tree.
- Believing MST applies to directed graphs.
- Forgetting that an MST always has **V − 1 edges**.
- Confusing MST with the shortest path problem.

---

# Best Practices

- Always verify that the graph is **connected** before attempting to find an MST.
- Check for cycles when adding edges.
- Use adjacency lists for sparse graphs and adjacency matrices for dense graphs when implementing algorithms.
- Draw the graph during manual dry runs to visualize connectivity.
- Remember the formula: **A tree with V vertices always has V − 1 edges.**

# Summary

- A **Minimum Spanning Tree (MST)** is a spanning tree with the minimum possible total edge weight.
- An MST connects **all vertices**, contains **no cycles**, and has exactly **V − 1** edges.
- A graph may have multiple spanning trees, but only one or more with the minimum total weight.
- MST problems arise in many real-world network design scenarios where cost optimization is essential.
- The two most widely used algorithms for constructing an MST are **Kruskal's Algorithm** and **Prim's Algorithm**, both based on the **Greedy Strategy**.
- In the next lecture, we will study **Kruskal's Algorithm** in detail, including the **Disjoint Set (Union-Find)** data structure, complete dry runs, correctness intuition, complexity analysis, and implementations in both Java and C++.
