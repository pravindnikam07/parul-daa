# 1. Real World Motivation

Imagine a telecom company wants to connect **6 cities** with fiber-optic cables.

Each possible cable has a different installation cost.

Goal:

- Connect every city.
- Spend the least possible money.
- Avoid unnecessary cable loops.

If engineers randomly choose cables, the project may become unnecessarily expensive.

Instead, they follow a simple strategy:

> "Always install the cheapest available cable, provided it doesn't create a loop."

This is the idea behind **Kruskal's Algorithm**.

# 4. Definition

**Kruskal's Algorithm** is a Greedy Algorithm that constructs a Minimum Spanning Tree by repeatedly selecting the **minimum weight edge** that **does not form a cycle**.

---

# 5. Core Idea

Think of Kruskal as follows:

```text
Collect every edge

↓

Sort edges

↓

Pick smallest edge

↓

Does it create a cycle?

      ↓

No → Select

Yes → Ignore

↓

Repeat

↓

Until V−1 edges selected
```

---

# 6. Why Sort the Edges?

Suppose the graph is

| Edge | Weight |
| ---- | -----: |
| AB   |      8 |
| AC   |      2 |
| BC   |      4 |
| CD   |      1 |

If we don't sort,

we may choose

```text
AB = 8
```

first,

which is clearly not optimal.

Sorting guarantees that

the cheapest available edge is always considered first.


# 10. The Biggest Question

Students usually ask:

> "How do we know whether an edge creates a cycle?"

Excellent question.

Consider:

```text
A ---- B

B ---- C
```

Now we add

```text
A ---- C
```

Result

```text
      A
     / \
    B---C
```

A cycle is formed.

So this edge must be rejected.

But how can a computer detect this efficiently?

Answer:

**Disjoint Set (Union-Find)**.

# 15. Java Implementation

> **Note:** This is a simplified implementation for learning purposes. In the next lecture, you can introduce optimizations such as **Path Compression** and **Union by Rank**.

```java
import java.util.*;

class Edge implements Comparable<Edge> {
    int source, destination, weight;

    Edge(int source, int destination, int weight) {
        this.source = source;
        this.destination = destination;
        this.weight = weight;
    }

    @Override
    public int compareTo(Edge other) {
        return this.weight - other.weight;
    }
}

public class KruskalMST {

    static int[] parent;

    static int find(int vertex) {
        if (parent[vertex] == vertex)
            return vertex;

        return find(parent[vertex]);
    }

    static void union(int u, int v) {
        int rootU = find(u);
        int rootV = find(v);
        parent[rootU] = rootV;
    }

    public static void main(String[] args) {

        int vertices = 4;

        Edge[] edges = {
            new Edge(0,2,2),
            new Edge(2,3,3),
            new Edge(0,1,4),
            new Edge(0,3,5),
            new Edge(1,3,6)
        };

        Arrays.sort(edges);

        parent = new int[vertices];

        for(int i=0;i<vertices;i++)
            parent[i]=i;

        int totalCost=0;

        System.out.println("Edges in MST:");

        int edgeCount=0;

        for(Edge edge:edges){

            if(find(edge.source)!=find(edge.destination)){

                union(edge.source,edge.destination);

                System.out.println(edge.source+" - "+edge.destination+
                                   " : "+edge.weight);

                totalCost+=edge.weight;

                edgeCount++;

                if(edgeCount==vertices-1)
                    break;
            }
        }

        System.out.println("Total Cost = "+totalCost);
    }
}
```

**Expected Output**

```text
Edges in MST:
0 - 2 : 2
2 - 3 : 3
0 - 1 : 4
Total Cost = 9
```

# 16. C++ Implementation

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

struct Edge {
    int source, destination, weight;

    bool operator<(const Edge &other) const {
        return weight < other.weight;
    }
};

vector<int> parent;

int find(int vertex) {
    if (parent[vertex] == vertex)
        return vertex;

    return find(parent[vertex]);
}

void unite(int u, int v) {
    int rootU = find(u);
    int rootV = find(v);
    parent[rootU] = rootV;
}

int main() {

    int vertices = 4;

    vector<Edge> edges = {
        {0,2,2},
        {2,3,3},
        {0,1,4},
        {0,3,5},
        {1,3,6}
    };

    sort(edges.begin(), edges.end());

    parent.resize(vertices);

    for(int i=0;i<vertices;i++)
        parent[i]=i;

    int totalCost=0;
    int edgeCount=0;

    cout<<"Edges in MST\n";

    for(auto edge:edges){

        if(find(edge.source)!=find(edge.destination)){

            unite(edge.source,edge.destination);

            cout<<edge.source<<" - "<<edge.destination
                <<" : "<<edge.weight<<endl;

            totalCost+=edge.weight;

            edgeCount++;

            if(edgeCount==vertices-1)
                break;
        }
    }

    cout<<"Total Cost = "<<totalCost;

    return 0;
}
```

# 17. Time Complexity

| Operation             |                                     Complexity |
| --------------------- | ---------------------------------------------: |
| Sorting edges         |                                 **O(E log E)** |
| Union-Find operations | Approximately **O(E α(V))** with optimizations |
| Overall               |                                 **O(E log E)** |

> Here, **E** is the number of edges and **V** is the number of vertices. The function **α(V)** (inverse Ackermann function) grows so slowly that it is practically constant.

---

# 18. Advantages

- Simple and intuitive.
- Produces an optimal MST.
- Efficient for **sparse graphs** (graphs with relatively few edges).
- Easy to implement with Union-Find.

---

# 19. Limitations

- Requires sorting all edges.
- Needs a cycle detection mechanism.
- Less efficient than Prim's algorithm on dense graphs.

---

# 20. Common Mistakes

- Forgetting to sort edges before processing.
- Adding an edge without checking for a cycle.
- Stopping before selecting **V − 1** edges.
- Confusing the number of edges (**V − 1**) with the number of vertices (**V**).
- Implementing Union-Find incorrectly.

---

# 21. Best Practices

- Always sort edges in non-decreasing order of weight.
- Use **Path Compression** and **Union by Rank** for an efficient Union-Find implementation.
- Stop as soon as **V − 1** edges have been selected.
- Test the algorithm on disconnected graphs (an MST does not exist; instead, you obtain a **Minimum Spanning Forest**).

# 25. Summary

- **Kruskal's Algorithm** is a Greedy algorithm that constructs a Minimum Spanning Tree by repeatedly selecting the **smallest-weight edge** that does **not** form a cycle.
- The algorithm processes edges in **sorted order** and uses the **Union-Find (Disjoint Set)** data structure to detect cycles efficiently.
- The algorithm terminates after selecting exactly **V − 1** edges.
- The dominant operation is sorting, giving an overall time complexity of **O(E log E)**.
- Kruskal's algorithm is especially efficient for **sparse graphs**.
- In the next lecture, we will study **Prim's Algorithm**, compare its strategy with Kruskal's, and learn when each algorithm is the better choice.
