# 1. Real-World Motivation

Imagine a city planning department wants to connect **five buildings** with underground fiber-optic cables.

The buildings are:

- Library
- Computer Lab
- Office
- Hostel
- Auditorium

The cost of laying cable between every pair of buildings is different.

Instead of connecting buildings randomly, the engineers decide:

> Start from one building and keep expanding the existing network by connecting the nearest unconnected building.

This is exactly the strategy used in **Prim's Algorithm**.

---

# 2. Everyday Analogy

Imagine you are exploring an unknown island.

You start at your campsite.

From your current position, you always walk along the **shortest available path** to reach a new location.

You do **not** jump to distant locations.

You simply keep expanding the explored region.

This is Prim's Algorithm.

Suppose the graph is:



```text

C → D = 3
```

Choose the smallest.

Continue until every vertex is included.

# 4. Definition

**Prim's Algorithm** is a Greedy Algorithm that constructs a Minimum Spanning Tree by starting from any vertex and repeatedly adding the **minimum-weight edge** that connects the current tree to a new vertex.

---

# 5. Core Idea

```text
Choose any starting vertex

↓

Add it to MST

↓

Find the smallest edge
connecting MST to an
unvisited vertex

↓

Add that edge

↓

Repeat

↓

Stop after V−1 edges
```

Notice:

Unlike Kruskal,

Prim always grows **one connected tree**.

---

# 6. How Prim Differs from Kruskal

Imagine building roads.

### Kruskal

Looks at the **entire country**.

Chooses the cheapest road anywhere.

---

### Prim

Starts from one city.

Expands outward.

Never builds disconnected pieces.

# 11. Priority Queue (Min Heap)

In a large graph,

finding the minimum edge every time using linear search is slow.

Instead,

Prim uses a **Priority Queue (Min Heap)**.

The priority queue always keeps the smallest edge at the top.

Example:

```
Heap

      2
     / \
    4   5
```

Top element

```text
2
```

comes out first.

This significantly improves performance.

---

# 12. Adjacency Matrix vs Adjacency List

## Adjacency Matrix

Example:

|     | A   | B   | C   | D   |
| --- | --- | --- | --- | --- |
| A   | 0   | 4   | 2   | 5   |
| B   | 4   | 0   | 7   | 6   |
| C   | 2   | 7   | 0   | 3   |
| D   | 5   | 6   | 3   | 0   |

Good for:

Dense graphs.

---

## Adjacency List

```text
A

↓

(B,4)

↓

(C,2)

↓

(D,5)
```

Good for:

Sparse graphs.

# 13. Java Implementation (Priority Queue)

```java
import java.util.*;

class Pair {
    int vertex;
    int weight;

    Pair(int vertex, int weight) {
        this.vertex = vertex;
        this.weight = weight;
    }
}

public class PrimMST {

    static final int V = 4;

    public static void prim(int[][] graph) {

        boolean[] visited = new boolean[V];

        PriorityQueue<Pair> pq =
                new PriorityQueue<>(Comparator.comparingInt(a -> a.weight));

        pq.offer(new Pair(0,0));

        int totalCost = 0;

        System.out.println("Edges selected:");

        while(!pq.isEmpty()){

            Pair current = pq.poll();

            int u = current.vertex;

            if(visited[u])
                continue;

            visited[u]=true;

            totalCost += current.weight;

            System.out.println("Visit Vertex : " + u +
                               " Weight : " + current.weight);

            for(int v=0; v<V; v++){

                if(graph[u][v]!=0 && !visited[v]){

                    pq.offer(new Pair(v,graph[u][v]));
                }
            }
        }

        System.out.println("Total Cost = "+totalCost);
    }

    public static void main(String[] args){

        int[][] graph={
                {0,4,2,5},
                {4,0,7,6},
                {2,7,0,3},
                {5,6,3,0}
        };

        prim(graph);
    }
}
```

# 14. C++ Implementation (Priority Queue)

```cpp
#include <iostream>
#include <vector>
#include <queue>

using namespace std;

const int V = 4;

void prim(vector<vector<int>> &graph){

    vector<bool> visited(V,false);

    priority_queue<pair<int,int>,
            vector<pair<int,int>>,
            greater<pair<int,int>>> pq;

    pq.push({0,0});

    int totalCost=0;

    while(!pq.empty()){

        auto current=pq.top();

        pq.pop();

        int weight=current.first;
        int vertex=current.second;

        if(visited[vertex])
            continue;

        visited[vertex]=true;

        totalCost+=weight;

        cout<<"Visit "<<vertex
            <<" Weight "<<weight<<endl;

        for(int i=0;i<V;i++){

            if(graph[vertex][i]!=0 && !visited[i]){

                pq.push({graph[vertex][i],i});
            }
        }
    }

    cout<<"Total Cost = "<<totalCost<<endl;
}

int main(){

    vector<vector<int>> graph={
        {0,4,2,5},
        {4,0,7,6},
        {2,7,0,3},
        {5,6,3,0}
    };

    prim(graph);

    return 0;
}
```

# 15. Time Complexity

| Implementation               |    Time Complexity |
| ---------------------------- | -----------------: |
| Adjacency Matrix             |          **O(V²)** |
| Adjacency List + Binary Heap |     **O(E log V)** |
| Fibonacci Heap (Advanced)    | **O(E + V log V)** |

For most practical applications, **Adjacency List + Binary Heap** is the preferred implementation.

---

# 16. Advantages

- Efficient for **dense graphs**.
- Does not require sorting all edges.
- Naturally grows a connected tree.
- Easy to implement using a priority queue.

---

# 17. Limitations

- Less efficient than Kruskal's algorithm on very sparse graphs.
- Requires the graph to be connected to obtain an MST.
- Priority queue implementation is more involved than the basic Kruskal approach.

---

# 18. Common Mistakes

- Selecting an edge that connects two already visited vertices.
- Forgetting to mark vertices as visited.
- Adding the same vertex multiple times from the priority queue.
- Assuming Prim's algorithm must always start from vertex 0 (it can start from any vertex in a connected graph).
- Confusing the minimum edge from the **current tree** with the globally minimum edge.

---

# 19. Best Practices

- Use an adjacency list with a priority queue for large sparse graphs.
- Ignore stale entries in the priority queue by checking whether the vertex is already visited.
- Keep track of the parent of each vertex if you need to print the actual MST edges.
- Verify that all vertices are visited; otherwise, the graph is disconnected.

# 23. Summary

- **Prim's Algorithm** constructs a Minimum Spanning Tree by **growing one connected tree** from an arbitrary starting vertex.
- At every step, it chooses the **minimum-weight edge** that connects the current MST to an unvisited vertex.
- Unlike Kruskal's algorithm, Prim's algorithm works with **vertices and the expanding tree**, not by globally sorting edges.
- A **Priority Queue (Min Heap)** enables an efficient implementation with time complexity **O(E log V)** using an adjacency list.
- Prim's algorithm is often preferred for **dense graphs**, whereas Kruskal's algorithm is generally more suitable for **sparse graphs**.
